### 2. Supabase Quiz Progress Tracking — Fully Wired Up

The integration described in `improvement_analysis/quiz-progress-tracking.md` is now live and working end-to-end.

**What's deployed:**
- `docs/assets/js/quiz-tracker.js` — intercepts quiz completions and syncs to Supabase
- `docs/admin.html` — teacher dashboard (GitHub OAuth, reads all rows via RLS policy)
- `mkdocs.yml` line 103 — `quiz-tracker.js` in `extra_javascript`

**Supabase project:** `wltmawdleuvcjxqtzkmj.supabase.co`
- `quiz_results` table exists with schema from the planning doc
- Row-level security policies are in place
- `teachers` table exists — add GitHub logins here to grant dashboard access
- GitHub OAuth provider is enabled in Supabase
- Redirect URL allowlist includes `https://*.app.github.dev/*` and `https://<org>.github.io/*`

**Key implementation corrections vs. the planning doc:**
- mkdocs-quiz (v1.6.3) stores data under `quiz_progress_<page-url>` keys (NOT `quiz-<id>-answers`)
- Value shape: `{ "quiz-0": { answered: bool, correct: bool, selectedValues: [...] }, ... }` (NO `submitted`/`score`/`total` fields — these are derived)
- `improvement_analysis/quiz-progress-tracking.md` has been updated to reflect the correct format
- OAuth callback handling (URL hash parsing → localStorage write) is done manually in `quiz-tracker.js` and `admin.html` since we use a minimal client with no SDK

**Auth flow summary:**
1. User clicks "Sign in to save progress" → redirects to Supabase → GitHub
2. GitHub redirects back with `#access_token=...` in URL hash
3. `handleOAuthCallback()` parses hash, fetches user from `/auth/v1/user`, writes session to localStorage, cleans URL
4. `initAuth()` reads session and updates the header button
5. On quiz submission, `syncResult()` fires and POSTs to Supabase `/rest/v1/quiz_results`

**Signed-in button:** white text, transparent background with white border (readable on dark Material header). Click to sign out (with `confirm()` dialog).

**Admin page (`docs/admin.html`):**
- Uses anon key + GitHub OAuth (NOT service role key — safe to deploy publicly)
- Teachers table RLS policy grants read-all access to logins in the `teachers` table
- To add teachers: Supabase Table Editor → `teachers` → Insert row (github_login column)
